2.  概述
为有效管控第三方应用合理使用本平台接口，特增加应用程序签名校验拦截。未通过拦截将无法访问本平台接口。
3.  接口拦截范围
结合平台现状，目前仅针对用户登录、用户注册相关接口进行拦截：
接口 名称 说明
/login/** 用户登录服务 该服务下除/login/logout外 均拦截
/app/scan/qrcode/login 二维码登录确认
/user/register/cellphone 手机注册账户
/user/register/email 邮箱注册账户
/user/create/account 创建下级用户账号
/user/send/sms/captcha 发送手机验证码
/user/send/email/captcha发送邮箱验证码
/user/reset/password 找回密码
/deviceManufacturer/add创建厂家
4.  签名算法
4.1  前置条件
第三方应用需在本平台申请 AppID 和 AppSecret ，妥善保管、合理使用。如因私自对外泄露 AppID 、 AppSecret 等信息导致数据丢失或服务终止等情况责任自
负。
4.2  签名步骤
4.2.1  公共参数
参数 名称 说明
IOT-Open-AppID AppID 本平台申请的AppID
IOT-Open-Nonce 随机数 长度为32的随机字符串，随机性越大越好，每个请求都需要重新生成。15分钟内不允许重复。
IOT-Open-Body-HashHttp Body 哈希值
对于GET请求，该值为空字符串；非GET类请求（如POST，Content-Type:application/json;charset=UTF-8），该值计算方法为sha256(httpBody).toLowerHexText()
4.2.2 URL 参数
（ 1 ）忽略   与  4.2.1  公共参数   中同名的参数；
（ 2 ）参数值必须为原始值，即不做任何处理，如 URL 编码等
4.2.3  合并参数并排序
合并  4.2.1  公共参数   与  4.2.2 URL 参数，并按照参数名排序。
4.2.4  拼接合并后的参数
将排序后的参数键值对用 & 拼接，即拼接成 key1=val1&key2=val2&...
4.2.5  对拼接后的字符串进行 Base64

字符集为 Utf8
4.2.6  对 Base64 后的字符串计算 Hash
Hash 算法为 HMAC­SHA256 ， Hash Key 为   本平台申请的  AppSecret  。
4.2.7  对 Hash 后的原始数据进行 md5 ，即为签名值
5.  使用签名
对于需要签名的接口，在 Htpp Header 中增加如下参数
参数 名称 说明
IOT-Open-AppID AppID 与参与计算签名的AppID值一致
IOT-Open-Nonce 随机数 与参与计算签名的Nonce值一致
IOT-Open-Sign签名 4.2.6 计算后的签名值
